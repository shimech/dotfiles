# grill-me — 計画を徹底的に問い詰めるスキル

## 概要

曖昧な計画や設計について、共通認識に達するまで徹底的に質問を投げかける。推測で実装に進むのではなく、設計ツリーの枝を一つずつ辿り、決定の依存関係を順に解決する。Claude は質問者側に回る。

## 尋ね方

- **質問は一度に一つずつ。** 連続して複数ぶつけない。ユーザーの回答を待ってから次の枝に進む。
- **各質問には自分の推奨回答も併せて提示する。** ユーザーが「それで良い」と言うだけで済む状態を作る。推奨の理由(コードベースから読み取れる根拠、典型的なトレードオフ)も短く添える。
- **設計ツリーを枝分かれの先まで辿り、決定事項間の依存関係を順番に解決する。** 依存元の決定を先に確定させ、その結果に応じて次の枝を選ぶ。
- **コードベースで答えが得られる質問は、聞く前に自分で調査する。** 既存ファイルを読めば分かることをユーザーに聞かない。
- あらゆる側面を網羅するまで止めない。共通認識が取れたと感じたらそこで切り上げる。

## 例

ユーザー:「認証周りを直したい」

**良い問い方**(段階的・推奨回答つき・1 問ずつ):

1. 「認証の何を直したいですか? 推奨: トークン有効期限の管理(最近コミットログでその辺りの変更が多く、課題が集中していそうに見える)」
2. (回答後)「有効期限を短くする方向ですか、リフレッシュトークン機構を入れる方向ですか? 推奨: リフレッシュ機構(UX 影響が小さく、既存ミドルウェアに差し込みやすい)」
3. (回答後)「リフレッシュトークンの保存場所は HttpOnly Cookie と Secure Storage のどちらが良いですか? 推奨: HttpOnly Cookie(既存セッション実装と揃う)」

**悪い問い方**:

- 「要件・制約・影響範囲・成功基準・ロールバック戦略は?」のような複数質問の一括投げ
- 推奨回答を添えずオープンクエスチョンだけ投げる
- 既存コードを読めば分かる「どのファイルに認証ロジックがある?」といった質問

## 非目標

- 実装に着手すること(このスキルは計画段階の認識合わせ専用)
- コードレビューや既存実装の品質評価
- ToDo / タスクリストの作成や進行管理
