# Rethink

今の自分の思考が歪んでいないかを検査し、歪みがあれば外す。現在の案と代替案を、どちらか一方に寄らずフラットに評価し直す。

## 大原則: 逆バイアスを避ける

「考え直して」と言われたとき、Claude は「今の案を否定しないといけない」という逆向きのバイアスに陥りやすい。再評価の結果、**現案がそのまま最善**も正当な結論。「念のため変える」「無難に代替案を採る」は結論にしない。

## バイアス検査と外し方

以下を一つずつ自問し、該当したら外し方に従ってその箇所**だけ**書き直す（全部ひっくり返さない）。

- **アンカリング**: 過去の自分の提案・発言との辻褄合わせ、最初に思いついた案への固執、セッション中に作った変更を守ろうとする動き。
  → 経緯を全部捨て、この流れを知らない第三者の目で「今の最終状態」だけを見る。
- **逆バイアス / 過剰修正**: ユーザーが疑問を出したことを根拠に現案を否定していないか。「変えるべき理由」をひねり出していないか。
  → 「ユーザーが疑問を持った」という事実を根拠から切り離す。現案の強みは強みとして素直に書く。
- **会話バイアス**: ユーザーの言葉・前提・問いの枠組みを無批判に受け入れていないか。会話の方向に合わせて情報を取捨選択していないか。
  → ユーザーの発言を「仮説」として扱い直す。問いの枠組み自体を疑う。
- **迎合**: 言いにくい指摘の省略、望まれそうな結論への寄せ、「まあいいか」で流した点。
  → 避けている指摘を明示し、改善案とセットで出す。

## 比較

代替案がある、または出せる場合は、片方だけ厚く書かない。A の欠点から B のメリットを逆算しない。
結論は「A が依然最善」「B に切り替え」「条件次第」のどれでもよく、根拠で勝負する。
